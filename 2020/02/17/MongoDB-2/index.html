MongoSQL-2

创建并连接

  • 创建并选择数据库

    use TEST1

  • 创建集合

    db.createCollection("SET1", { capped : true, autoIndexId : true, size : 6142800, max : 10000} )

向集合内插入两个文档

  • 定义变量:

A1=({ A: ['1', '2', '3'], B: ['1', '2', '3'], C: ['1', '2', '3'], D: ['1', '2', '3'], E: ['1', '2', '4'], likes: 100});

  • 插入文档

db.SET1.find()

  • 更新数据

    db.SET1.updateMany({'E': ['1' , '2', '4']}, {$set:{'E': ['1', '2', '3']}},{multi:true})

  • 查看集合内容

    db.SET1.find().pretty()

  • 查看数据库

    db

  • 查看所有数据库

    show dbs

创建索引

  • 因为索引不能用于并行数组
  • 所以这里新建了一个文档并进行字符串插入
  • 将文档SET2内的键A和键分别设置为升序索引与降序索引(DRBMS内称之为复合索引)

    db.SET2.createIndex({"A":1,"B":-1})

  • 查看所有索引

    db.SET2.getIndexes()

  • 删除所有索引

    db.SET2.dropIndexes()

  • 删除集合SET1内所有符合条件的文档

    db.SET1.deleteMany({A: ['1', '2', '3']})
    db.SET1.remove({'A':['1', '2', '3']})

  • 查看集合内容

    db.SET1.find().pretty()

  • 删除集合

    db.SET1.drop()

  • 删除所在数据库

    db.dropDatabase()

  • 查看已有数据库列表

    show dbs

补充内容:

  • 文档的数据结构和JOSN基本相同
  • 所有存储在集合中的数据都是BSON格式
  • BSON 是一种类似 JSON 的二进制形式的存储格式，是 Binary JSON 的简称
  • 增删改查中只有查和改有实际用途，而精髓就在于查
